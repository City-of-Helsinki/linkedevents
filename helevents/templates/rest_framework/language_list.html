Using the language endpoint

Here, languages for events are listed. In the event endpoint, the event languages may be shown in the in_language field, if known.

Also, event data may have translations in the languages which have translation_available set to true. Currently, translations in fi, sv, en, ru, zh_hans, and ar are supported.

Events organized in or translated to a specific language can be found at the event endpoint using the query parameter language.

Language as service language

To find out languages which can or cannot be used as service language in a registration, use the query parameterservice_language.

Example:

event/?service_language=true
